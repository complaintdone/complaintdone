## Project Overview

ComplaintDone is a SaaS application that generates AI-powered complaint letters. Users fill out a form with their complaint details, pay £1.49 via Stripe, and receive a professionally formatted complaint letter generated by Claude AI (Anthropic).

**Tech Stack:**
- Frontend: React 18 + TypeScript + Vite
- UI: shadcn/ui components + Tailwind CSS
- Backend: Supabase Edge Functions (Deno runtime)
- Payment: Stripe Checkout
- AI: Anthropic Claude API (claude-haiku-4-5-20251001)
- Routing: React Router v6

## Supabase Edge Functions

Located in `supabase/functions/`. Each function runs in Deno runtime and requires specific environment variables.

### Deploying Functions

Functions are configured in `supabase/config.toml`. All functions have `verify_jwt = true` for authentication.

**Available Functions:**

1. **create-checkout** - Creates Stripe checkout session
   - Requires: `STRIPE_SECRET_KEY`
   - Input: `{ email, name, company, description, tone }`
   - Returns: `{ url }` (Stripe checkout URL)

2. **generate-complaint** - Generates complaint letter via Claude API
   - Requires: `ANTHROPIC_API_KEY`
   - Input: `{ name, company, description, tone }`
   - Returns: `{ letter }` (formatted complaint text)

3. **send-email** - Sends generated complaint letter to user
   - Purpose: Email delivery after payment success

4. **stripe-webhook** - Handles Stripe payment webhooks
   - Purpose: Process payment completion events

### Environment Variables Required

```
STRIPE_SECRET_KEY=sk_test_...
ANTHROPIC_API_KEY=sk-ant-...
VITE_SUPABASE_URL=https://...supabase.co
VITE_SUPABASE_ANON_KEY=eyJ...
```

### Key Flow

1. User fills form at `/complaint` (Complaint.tsx:65-95)
2. Form submits to `create-checkout` function with complaint data
3. Stripe Checkout session created with metadata containing complaint details
4. User redirected to Stripe payment page
5. On success, Stripe webhook triggers `stripe-webhook` function
6. Webhook calls `generate-complaint` to create letter via Claude API
7. Letter sent to user via `send-email` function
8. User redirected to `/success`

### Important Implementation Details

**Complaint Form (src/pages/Complaint.tsx):**
- 2000 character limit on description
- Pre-defined list of UK companies with autocomplete
- Three tone options: polite, firm, assertive
- Form validation requires email, company, description
- Calls create-checkout function which returns Stripe URL

**Supabase Client:**
- Uses localStorage for session persistence
- Auto-refresh tokens enabled
- Imported as: `import { supabase } from "@/integrations/supabase/client"`

**Styling:**
- Uses CSS custom properties for theming (defined in src/index.css)
- Custom color palette: navy, orange, with semantic tokens
- Font families: 'Space Grotesk' (headings), 'DM Sans' (body)
- Component variants defined via class-variance-authority

**Routing:**
- React Router v6 with BrowserRouter
- Path imports use `@/` alias (resolves to `./src`)

## Common Issues

**Supabase Functions:**
- Functions run in Deno, not Node.js - use Deno-compatible imports (e.g., `https://esm.sh/` for npm packages)
- CORS headers required on all function responses
- Always handle OPTIONS preflight requests

**Environment Variables:**
- Frontend vars must be prefixed with `VITE_`
- Backend function vars set in Supabase dashboard or local .env

**Stripe Integration:**
- Hardcoded price ID: `price_1T7w2IPt9nNFZaKHsb5S6GUv` (£1.49)
- Success/cancel URLs point to production domain: `complaintdone.com`
- Metadata in checkout session carries complaint details to webhook

**AI Generation:**
- Model: claude-haiku-4-5-20251001 (fast, cost-effective)
- Max tokens: 1024
- Prompt structure in generate-complaint/index.ts:37-41
