## Project Overview

ComplaintDone is a SaaS application that generates AI-powered complaint letters. Users fill out a form with their complaint details, pay £3 (UK) or $5 (USA) via Stripe, and receive a professionally formatted complaint letter generated by Claude AI (Anthropic).

**Tech Stack:**
- Frontend: React 18 + TypeScript + Vite
- UI: shadcn/ui components + Tailwind CSS
- Backend: Supabase Edge Functions (Deno runtime)
- Payment: Stripe Checkout
- AI: Anthropic Claude API (claude-haiku-4-5-20251001)
- Email: Resend API
- Database: PostgreSQL (Supabase managed)
- Routing: React Router v6

## Supabase Edge Functions

Located in `supabase/functions/`. Each function runs in Deno runtime and requires specific environment variables.

### Deploying Functions

Functions are configured in `supabase/config.toml`. **Currently all functions have `verify_jwt = false`** (security gap - see SECURITY_HANDOFF.md).

**Deploy Command:**
```bash
supabase functions deploy <function-name>
```

**Available Functions:**

1. **create-checkout** - Creates Stripe checkout session with rate limiting
   - Requires: `STRIPE_SECRET_KEY`, `SUPABASE_SERVICE_ROLE_KEY`
   - Input: `{ email, name, company, description, tone }`
   - Returns: `{ url }` (Stripe checkout URL)
   - **Security:** Rate limited to 5 requests per IP per 60 minutes (index.ts:16-68)
   - **Details:** Splits descriptions into 490-char chunks for Stripe metadata

2. **generate-complaint** - Generates complaint letter via Claude API
   - Requires: `ANTHROPIC_API_KEY`
   - Input: `{ name, company, description, tone }`
   - Returns: `{ letter }` (formatted complaint text)
   - **Model:** claude-haiku-4-5-20251001 (fast, cost-effective)

3. **send-email** - Sends generated complaint letter to user
   - Requires: `RESEND_API_KEY`
   - Input: `{ name, email, company, letter }`
   - Returns: Email delivery confirmation
   - **From:** noreply@complaintdone.com

4. **stripe-webhook** - Handles Stripe payment webhooks
   - Requires: `STRIPE_WEBHOOK_SECRET`, `SUPABASE_URL`, `SUPABASE_ANON_KEY`
   - **Security:** HMAC-SHA256 signature verification (index.ts:10-62)
   - **Details:** Reassembles chunked descriptions, calls generate-complaint + send-email

### Key Flow

1. User fills form at `/complaint` (Complaint.tsx:65-95)
2. Form submits to `create-checkout` function with complaint data
3. **Rate limiting check:** IP address checked against rate_limits table (5/hour limit)
4. Stripe Checkout session created with metadata containing complaint details
5. User redirected to Stripe payment page
6. On success, Stripe sends webhook event to `stripe-webhook` function
7. **Signature verification:** Webhook validates HMAC-SHA256 signature (prevents abuse)
8. Webhook reassembles chunked description from metadata
9. Webhook calls `generate-complaint` to create letter via Claude API
10. Letter sent to user via `send-email` function
11. User redirected to `/success`

## Common Issues

**Supabase Functions:**
- Functions run in Deno, not Node.js - use Deno-compatible imports (e.g., `https://esm.sh/` for npm packages)
- CORS headers required on all function responses
- Always handle OPTIONS preflight requests
- **IMPORTANT:** Stripe SDK's `constructEvent` crashes in Deno (EarlyDrop error) - manual HMAC verification implemented using `crypto.subtle` API
- Use `SUPABASE_SERVICE_ROLE_KEY` to bypass RLS when needed (e.g., rate limiting)
- After deployment, functions may take 1-2 minutes to update (cold start)

**Environment Variables:**
- Frontend vars must be prefixed with `VITE_`
- Backend function vars set in Supabase dashboard or local .env

**Stripe Integration:**
- Price IDs configured via environment variables (£3 UK / $5 USA)
- Success/cancel URLs point to production domain: `complaintdone.com`
- Metadata in checkout session carries complaint details to webhook
- Metadata limit: 500 chars per value - descriptions split into chunks if >490 chars

**AI Generation:**
- Model: claude-haiku-4-5-20251001 (fast, cost-effective)
- Max tokens: 1024
- Prompt structure in generate-complaint/index.ts:37-41
- Cost: ~£0.001 per letter

**Database:**
- PostgreSQL managed by Supabase
- Migrations located in `supabase/migrations/`
- Apply with: `supabase db push --linked`
- Current tables:
  - `rate_limits` - IP-based rate limiting (ip, created_at)
- Cleanup function available: `cleanup_old_rate_limits()` (deletes records >24h old)

## 🔒 Security Status

### ✅ Implemented Protections

1. **Stripe Webhook Signature Verification** ✅ FIXED (8 March 2026)
   - Native HMAC-SHA256 verification using `crypto.subtle` API
   - Prevents unauthorized webhook calls
   - Replay attack prevention (5-minute timestamp tolerance)
   - Timing-safe comparison to prevent timing attacks
   - **Code:** `supabase/functions/stripe-webhook/index.ts:10-62`

2. **Rate Limiting** ✅ FIXED (8 March 2026)
   - 5 requests per IP per 60 minutes
   - Database-backed using `rate_limits` table (no external service costs)
   - Returns 429 status with `Retry-After: 3600` header
   - Graceful error handling (continues on DB errors to avoid blocking legitimate users)
   - **Code:** `supabase/functions/create-checkout/index.ts:16-68`

3. **Input Validation**
   - Email format validation on frontend
   - 2000 character description limit
   - Required fields enforced (email, company, description)
   - **Code:** `src/pages/Complaint.tsx:65-95`

### ⚠️ Known Security Gaps (High Priority)

**See SECURITY_HANDOFF.md for complete details and implementation plans.**

1. **JWT Verification Disabled on All Functions**
   - All 4 functions have `verify_jwt = false` in config.toml
   - Dashboard setting "Verify JWT with legacy secret" = OFF
   - **Impact:** Functions accept unauthenticated requests
   - **Fix:** Re-enable for create-checkout, generate-complaint, send-email
   - **Note:** Keep disabled for stripe-webhook (Stripe doesn't send JWT)
   - **Priority:** HIGH - Should be fixed before scaling

2. **No Database Logging of Complaints**
   - No record of generated complaints
   - Cannot track revenue vs. complaints ratio
   - No customer support lookup capability
   - **Fix:** Create `complaints` table and log after successful generation
   - **Priority:** MEDIUM

3. **No Error Monitoring Service**
   - Only console.log (viewable in Supabase logs)
   - No alerts for production errors
   - **Fix:** Add Sentry for Deno
   - **Priority:** MEDIUM

4. **Hardcoded Values**
   - Stripe price ID hardcoded in create-checkout/index.ts:108
   - Success/cancel URLs hardcoded
   - **Fix:** Move to environment variables
   - **Priority:** LOW
